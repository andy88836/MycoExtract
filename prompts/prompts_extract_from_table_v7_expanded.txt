## 🚨 CRITICAL: PARAMETER EXTRACTION RULES

### Rule 1: STANDARD PARAMETER RECOGNITION
Extract into standard fields based on biochemical meaning, not just literal text matching:

| Standard Field | Accept these names and equivalents |
|----------------|------------------------------------|
| Km_value | Km, KM, K_m, Michaelis constant |
| kcat_value | kcat, k_cat, turnover number, **Vmax/E0**, **Vmax/[E0]**, **V_max/E_0** |
| kcat_Km_value | kcat/Km, k_cat/K_m, catalytic efficiency, Vmax/(E0×Km), Vmax/(E0·Km) |

**🔬 BIOCHEMICAL CONTEXT AWARENESS:**
- **Vmax/E0 = kcat**: According to Michaelis-Menten kinetics, kcat (turnover number) equals Vmax divided by enzyme concentration.
  - When you see "Vmax/E0", "Vmax/[E0]", or "V_max/E_0" → Extract as **kcat_value**
  - This is the enzyme's intrinsic catalytic rate per active site
- **kcat/Km = catalytic efficiency**: Can also be expressed as Vmax/(E0×Km)
  - Extract as **kcat_Km_value**

### Rule 2: REJECT TRULY NON-STANDARD HEADERS
DO NOT extract into standard fields if the header contains:

**Non-standard divisions:**
- Per protein mass: `/mg protein`, `/μmol enzyme`
  - Examples: ❌ "Km per mg protein", "activity/mg"
- Per other parameters: `/Ki`, `/[S]`
  - Examples: ❌ "kcat/Ki", "Vmax/[S]"
- Multiplication with substrates:
  - Examples: ❌ "kcat·[S]", "Km×[I]"
- Complex nested ratios (not standard biochemical parameters):
  - Examples: ❌ "(kcat/Km)/Ki", "Vmax²/Km"

**Modifiers/qualifiers:**
- Subscripts: `,app`, `,obs`, `,apparent`, `,relative`
  - Examples: ❌ "Km,app", "kcat,obs", "Vmax,relative"
- Conditions: `at pH X`, `(temperature)`, `in buffer`
  - Examples: ❌ "Km at pH 7", "kcat (37°C)"

**Normalization (non-intrinsic parameters):**
- Per protein mass: `/mg`, `/μmol`, `/mg protein`
- Per volume: `/mL`, `/L`

### Rule 3: HANDLE NON-STANDARD DATA
When you encounter a truly non-standard column (after applying biochemical recognition):
1. Leave the corresponding standard field as `null`
2. Record the original data in `notes`:
   ```
   "notes": "Table shows [column_name] = [value] [unit]. Not standard [parameter_name]."
   ```
3. If multiple non-standard columns exist, concatenate in notes

**Example of NON-STANDARD (truly cannot map):**
- Table column: "Activity per mg protein (U/mg)" with value "45.2"
- Extract as:
  - `kcat_value: null`
  - `notes: "Table shows Activity per mg protein = 45.2 U/mg. Requires enzyme concentration for kcat calculation."`

**Example of STANDARD (biochemically mappable):**
- Table column: "Vmax/E0 (min⁻¹)" with value "27.1"
- Extract as:
  - `kcat_value: 27.1`
  - `kcat_unit: "min⁻¹"`
  - ✅ This IS standard kcat (turnover number)

### Rule 4: CONSERVATIVE PRINCIPLE
**"When in doubt, leave it out."**
- Uncertain if header matches? → Leave as `null`, document in `notes`
- Better to miss data than create false mappings
- Human reviewers can interpret `notes` and add correct values later

## Temperature and Stability Extraction

1. **Optimal Temperature Inference**:
   - If table lists **Activity vs Temperature**:
     - Identify the temperature with the HIGHEST activity.
     - Extract this as `optimal_temperature_value`.
     - Do NOT extract every row as a separate record unless they represent different substrates/enzymes.
     - If the table is JUST for determining optimums, summarize it into one record with `optimal_temperature_value`.

2. **Thermal Stability Extraction**:
   - If table lists **Residual Activity vs Temperature** (after incubation):
     - This is a stability test.
     - Extract the highest temperature where activity is maintained (or the range) into `thermal_stability`.
     - Look for incubation time in caption (e.g., "incubated for 1h") -> `thermal_stability_time`.
